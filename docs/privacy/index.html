Privacy Policy for LeetCode Repetition

Last updated: May 31, 2025

Note: LeetCode Repetition is not affiliated with, endorsed by, or sponsored by LeetCode, LLC.

This Privacy Policy explains how LeetCode Repetition (“we,” “us,” or “our”) collects, uses, stores, and shares:

  • a unique identifier from Google Sign-In (the “Google User ID”);
  • LeetCode session cookies for obtaining the user’s LeetCode username and LeetCode user ID;
  • no other personal data or browsing activity.

We collect these values to recognize users across sessions, store them in a Supabase SQL table, and retain them until a user requests deletion. All data transmissions occur over HTTPS. You can revoke Google Sign-In access or uninstall the extension at any time, and if you sign out of LeetCode, we will stop using your LeetCode session cookies.

1. Introduction

LeetCode Repetition (“we,” “us,” or “our”) is a browser extension that helps users revisit challenging LeetCode problems using spaced-repetition techniques. When you install or use our extension, we collect only (1) a unique Google User ID, and (2) your LeetCode username and LeetCode user ID via your active LeetCode session cookies. We do not collect any additional profile or credential information.

2. Information We Collect

  • Google User ID
    Upon your first Google Sign-In, we request only the openid scope to obtain the “sub” field from the ID token, which is your Google User ID.
  • LeetCode Session Cookies
    When you choose to sync your LeetCode account, our extension reads the LeetCode session cookies you already have. We use these cookies only to fetch from LeetCode’s API:
    • your LeetCode username
    • your LeetCode user ID

3. How We Use Your Information

  • User Identification & Syncing
    We use the Google User ID to identify you in our Supabase database so that your settings, progress, and preferences can sync across devices.
  • LeetCode Profile Fetching
    We use your LeetCode session cookies to fetch your username and user ID so you can import your existing problem list and track solved problems.

4. Data Retention and Storage

  • Supabase SQL Table
    Your Google User ID and LeetCode user ID are stored in Supabase. Data is encrypted at rest (AES-256) and in transit (TLS).
  • Retention Period
    We retain your Google User ID and LeetCode user ID until you request deletion.

5. Cookies and Tracking Technologies

We do not use cookies or tracking technologies beyond reading your LeetCode session cookies to fetch your username and user ID. Extension settings (e.g., review preferences) are saved in browser storage accessible only to this extension.

6. Data Security

  • HTTPS Encryption
    All communications with Google and LeetCode APIs occur over HTTPS (TLS encryption).
  • Database Security (Supabase)
    Supabase encrypts data at rest (AES-256) and in transit (TLS).
  • Local Browser Storage Protection
    If you sign out of Google or LeetCode, your Google User ID becomes inaccessible, and LeetCode session cookies are invalidated. Local extension data is cleared on uninstall.

7. Third-Party Services

  • Google Sign-In (OAuth 2.0)
    We use Google’s OAuth 2.0 API to obtain an ID token containing your Google User ID. Only the openid scope is requested.
  • LeetCode API
    We use your LeetCode session cookies to call LeetCode’s API and fetch your username and user ID.
  • Supabase (Database Backend)
    We store your Google User ID and LeetCode user ID in Supabase. Visit https://supabase.com/privacy for details.

8. Your Choices and Rights

  • Revoke Google Sign-In
    To revoke access, go to your Google Account’s “Security” → “Third-party apps with account access” and remove LeetCode Repetition.
  • Uninstall the Extension
    Uninstalling removes your Google User ID from local storage; Supabase retains it until you request deletion.
  • Revoke LeetCode Access
    To revoke access, clear your LeetCode session cookies or sign out of LeetCode in your browser.
  • Data Deletion (Supabase)
    To delete your data from Supabase, email leetcode-repetition@murrah.dev with “Data Deletion Request” in the subject.
  • Access and Correction
    You may request a copy or correction of your stored data by contacting us at leetcode-repetition@murrah.dev.

9. Changes to This Privacy Policy

We may update this Privacy Policy to reflect changes in our practices or legal requirements. When material changes occur, we will update the “Last updated” date at the top and notify you via an in-extension banner or release notes.
